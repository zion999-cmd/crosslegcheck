# STM32开发工具链完整说明

## 🛠️ 您已安装的ST工具

### 1. STM32CubeMX 🎨 (配置工具)
- **作用**: 图形化芯片配置
- **功能**: 
  - 配置引脚分配
  - 配置外设参数
  - 生成初始化代码
  - 管理固件库
- **使用场景**: 项目初始化、修改硬件配置

### 2. STM32CubeIDE 🔧 (开发环境)
- **作用**: 完整的集成开发环境
- **功能**:
  - 包含CubeMX功能
  - **代码编辑**
  - **项目编译** ⭐️
  - **程序调试**
  - **固件烧录** (通过调试器)
- **使用场景**: 日常开发、编译、调试

### 3. STM32CubeProgrammer 📱 (烧录工具)
- **作用**: 专业的固件烧录和芯片操作工具
- **功能**:
  - **独立烧录固件** ⭐️ (不需要IDE)
  - 读取/擦除Flash
  - 读取芯片信息
  - 设置Option Bytes (配置字节)
  - 支持多种接口：ST-Link, UART, USB DFU, SPI, I2C
  - **批量生产烧录**
- **使用场景**: 生产烧录、维修、芯片恢复

## 🎯 工具链工作流程

### 开发阶段流程
```
STM32CubeMX → STM32CubeIDE → STM32CubeProgrammer
    ↓              ↓                ↓
  配置芯片      编译+调试         生产烧录
```

### 具体使用场景

#### 🔄 **日常开发** (使用CubeIDE)
```
编写代码 → 编译 → 调试 → 烧录测试
      ↑__________________|
     (全部在CubeIDE中完成)
```

#### 🏭 **生产部署** (使用CubeProgrammer)
```
编译好的.hex/.bin文件 → CubeProgrammer → 批量烧录到产品
```

#### 🚨 **故障恢复** (使用CubeProgrammer)
```
芯片锁死/损坏 → CubeProgrammer → 恢复/重新配置
```

### 实际应用示例

```bash
# 在CubeIDE中一键完成：
编译 → 连接ST-Link → 自动烧录 → 启动调试
```

#### 场景2: 独立烧录 (使用CubeProgrammer)

#### 场景3: 芯片恢复 (CubeProgrammer专长)
```bash
# 芯片被意外锁定时：
CubeProgrammer → 连接芯片 → 解锁/擦除 → 重新烧录
```

## 🎯 您的STM32H750项目

### 推荐工作流程
1. **现在**: 用STM32CubeIDE编译和调试
2. **调试完成后**: 可用CubeProgrammer进行最终烧录验证
3. **如果有硬件问题**: CubeProgrammer可以诊断芯片状态

### CubeProgrammer的额外价值
- **芯片信息读取**: 确认芯片型号、Flash大小等
- **Option Bytes配置**: 设置启动方式、看门狗等
- **Flash保护设置**: 保护固件不被读取
- **多接口支持**: 如果ST-Link不可用，可用UART等其他方式
